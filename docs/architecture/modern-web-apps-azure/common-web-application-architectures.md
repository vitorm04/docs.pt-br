## <a name="what-are-layers"></a><span data-ttu-id="0da05-133">O que são camadas?</span><span class="sxs-lookup"><span data-stu-id="0da05-133">What are layers?</span></span>

<span data-ttu-id="0da05-134">Conforme a complexidade dos aplicativos aumenta, uma maneira de gerenciar essa complexidade é dividir o aplicativo de acordo com suas responsabilidades ou interesses.</span><span class="sxs-lookup"><span data-stu-id="0da05-134">As applications grow in complexity, one way to manage that complexity is to break up the application according to its responsibilities or concerns.</span></span> <span data-ttu-id="0da05-135">Isso segue a separação do princípio de interesses e pode ajudar a manter organizada uma base de código em expansão, de modo que os desenvolvedores possam encontrar com facilidade o local em que determinada funcionalidade foi implementada.</span><span class="sxs-lookup"><span data-stu-id="0da05-135">This follows the separation of concerns principle, and can help keep a growing codebase organized so that developers can easily find where certain functionality is implemented.</span></span> <span data-ttu-id="0da05-136">Apesar disso, a arquitetura em camadas oferece inúmeras vantagens, além de apenas a organização do código.</span><span class="sxs-lookup"><span data-stu-id="0da05-136">Layered architecture offers a number of advantages beyond just code organization, though.</span></span>

<span data-ttu-id="0da05-139">Com uma arquitetura em camadas, os aplicativos podem impor restrições sobre quais camadas podem se comunicar com outras camadas.</span><span class="sxs-lookup"><span data-stu-id="0da05-139">With a layered architecture, applications can enforce restrictions on which layers can communicate with other layers.</span></span> <span data-ttu-id="0da05-140">Isso ajuda na obtenção do encapsulamento.</span><span class="sxs-lookup"><span data-stu-id="0da05-140">This helps to achieve encapsulation.</span></span> <span data-ttu-id="0da05-141">Quando uma camada é alterada ou substituída, somente as camadas que trabalham com ela devem ser afetadas.</span><span class="sxs-lookup"><span data-stu-id="0da05-141">When a layer is changed or replaced, only those layers that work with it should be impacted.</span></span> <span data-ttu-id="0da05-142">Ao limitar quais camadas dependem de outras camadas, o impacto das alterações pode ser reduzido, de modo que uma única alteração não afete todo o aplicativo.</span><span class="sxs-lookup"><span data-stu-id="0da05-142">By limiting which layers depend on which other layers, the impact of changes can be mitigated so that a single change doesn't impact the entire application.</span></span>
