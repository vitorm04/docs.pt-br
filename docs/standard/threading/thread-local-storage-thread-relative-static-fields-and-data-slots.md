# <a name="thread-local-storage-thread-relative-static-fields-and-data-slots"></a><span data-ttu-id="c2c62-102">Armazenamento local de thread: campos estáticos relativos a thread e slots de dados</span><span class="sxs-lookup"><span data-stu-id="c2c62-102">Thread Local Storage: Thread-Relative Static Fields and Data Slots</span></span>
<span data-ttu-id="c2c62-103">Você pode usar o armazenamento local de thread (TLS) gerenciado para armazenar dados que são exclusivos de um domínio de aplicativo e thread.</span><span class="sxs-lookup"><span data-stu-id="c2c62-103">You can use managed thread local storage (TLS) to store data that is unique to a thread and application domain.</span></span> <span data-ttu-id="c2c62-104">O .NET Framework oferece duas maneiras de usar o TLS gerenciado: campos estáticos relativos a thread e slots de dados.</span><span class="sxs-lookup"><span data-stu-id="c2c62-104">The .NET Framework provides two ways to use managed TLS: thread-relative static fields and data slots.</span></span>  
  
-   <span data-ttu-id="c2c62-105">Use campos estáticos relativos a thread (campos `Shared` relativos a thread no Visual Basic) se você puder prever suas necessidades exatas em tempo de compilação.</span><span class="sxs-lookup"><span data-stu-id="c2c62-105">Use thread-relative static fields (thread-relative `Shared` fields in Visual Basic) if you can anticipate your exact needs at compile time.</span></span> <span data-ttu-id="c2c62-106">Campos estáticos relativos a thread fornecem o melhor desempenho.</span><span class="sxs-lookup"><span data-stu-id="c2c62-106">Thread-relative static fields provide the best performance.</span></span> <span data-ttu-id="c2c62-107">Eles também oferecem os benefícios da verificação do tipo de tempo de compilação.</span><span class="sxs-lookup"><span data-stu-id="c2c62-107">They also give you the benefits of compile-time type checking.</span></span>  
  
-   <span data-ttu-id="c2c62-108">Use slots de dados quando seus requisitos reais podem ser descobertos apenas em tempo de execução.</span><span class="sxs-lookup"><span data-stu-id="c2c62-108">Use data slots when your actual requirements might be discovered only at run time.</span></span> <span data-ttu-id="c2c62-109">Os slots de dados são mais lentos e mais difíceis de usar do que os campos estáticos relativos a thread, e os dados são armazenados como tipo <xref:System.Object>, então você deve lançá-los no tipo correto antes de usá-los.</span><span class="sxs-lookup"><span data-stu-id="c2c62-109">Data slots are slower and more awkward to use than thread-relative static fields, and data is stored as type <xref:System.Object>, so you must cast it to the correct type before you use it.</span></span>  
  
 <span data-ttu-id="c2c62-110">No C ++ não gerenciado, você usa `TlsAlloc` para alocar slots dinamicamente e `__declspec(thread)` para declarar que uma variável deve ser alocada no armazenamento relativo ao thread.</span><span class="sxs-lookup"><span data-stu-id="c2c62-110">In unmanaged C++, you use `TlsAlloc` to allocate slots dynamically and `__declspec(thread)` to declare that a variable should be allocated in thread-relative storage.</span></span> <span data-ttu-id="c2c62-111">Os campos estáticos relativos a thread e os slots de dados fornecem a versão gerenciada desse comportamento.</span><span class="sxs-lookup"><span data-stu-id="c2c62-111">Thread-relative static fields and data slots provide the managed version of this behavior.</span></span>  
  
 <span data-ttu-id="c2c62-112">No [!INCLUDE[net_v40_long](../../../includes/net-v40-long-md.md)], você pode usar a classe <xref:System.Threading.ThreadLocal%601?displayProperty=nameWithType> para criar objetos locais de thread que são inicializados lentamente quando o objeto é consumido pela primeira vez.</span><span class="sxs-lookup"><span data-stu-id="c2c62-112">In the [!INCLUDE[net_v40_long](../../../includes/net-v40-long-md.md)], you can use the <xref:System.Threading.ThreadLocal%601?displayProperty=nameWithType> class to create thread-local objects that are initialized lazily when the object is first consumed.</span></span> <span data-ttu-id="c2c62-113">Para obter mais informações, veja [Inicialização lenta](../../../docs/framework/performance/lazy-initialization.md).</span><span class="sxs-lookup"><span data-stu-id="c2c62-113">For more information, see [Lazy Initialization](../../../docs/framework/performance/lazy-initialization.md).</span></span>  
  
## <a name="uniqueness-of-data-in-managed-tls"></a><span data-ttu-id="c2c62-114">Exclusividade de dados em TLS gerenciado</span><span class="sxs-lookup"><span data-stu-id="c2c62-114">Uniqueness of Data in Managed TLS</span></span>  
 <span data-ttu-id="c2c62-115">Não importa se você usa campos estáticos relativos a thread ou slots de dados, os dados em TLS gerenciado são exclusivos da combinação de thread e domínio de aplicativos.</span><span class="sxs-lookup"><span data-stu-id="c2c62-115">Whether you use thread-relative static fields or data slots, data in managed TLS is unique to the combination of thread and application domain.</span></span>  
  
-   <span data-ttu-id="c2c62-116">Dentro de um domínio de aplicativo, um thread não pode modificar dados de outro thread, mesmo quando ambos threads usam o mesmo campo ou slot.</span><span class="sxs-lookup"><span data-stu-id="c2c62-116">Within an application domain, one thread cannot modify data from another thread, even when both threads use the same field or slot.</span></span>  
  
-   <span data-ttu-id="c2c62-117">Quando um thread acessa o mesmo campo ou slot de vários domínios de aplicativos, um valor separado é mantido em cada domínio de aplicativo.</span><span class="sxs-lookup"><span data-stu-id="c2c62-117">When a thread accesses the same field or slot from multiple application domains, a separate value is maintained in each application domain.</span></span>  
  
 <span data-ttu-id="c2c62-118">Por exemplo, se um thread define o valor de um campo estático relativo a thread, entra em outro domínio do aplicativo e, em seguida, recupera o valor do campo, o valor recuperado no segundo domínio do aplicativo difere do valor no primeiro domínio do aplicativo.</span><span class="sxs-lookup"><span data-stu-id="c2c62-118">For example, if a thread sets the value of a thread-relative static field, enters another application domain, and then retrieves the value of the field, the value retrieved in the second application domain differs from the value in the first application domain.</span></span> <span data-ttu-id="c2c62-119">Definir um novo valor para o campo no segundo domínio do aplicativo não afeta o valor do campo no primeiro domínio do aplicativo.</span><span class="sxs-lookup"><span data-stu-id="c2c62-119">Setting a new value for the field in the second application domain does not affect the field's value in the first application domain.</span></span>  
  
 <span data-ttu-id="c2c62-120">Da mesma forma, quando um segmento obtém o mesmo slot de dados nomeado em dois domínios de aplicativo diferentes, os dados no primeiro domínio do aplicativo permanecem independentes dos dados no segundo domínio do aplicativo.</span><span class="sxs-lookup"><span data-stu-id="c2c62-120">Similarly, when a thread gets the same named data slot in two different application domains, the data in the first application domain remains independent of the data in the second application domain.</span></span>  
  
## <a name="thread-relative-static-fields"></a><span data-ttu-id="c2c62-121">Campos estáticos relativos a thread</span><span class="sxs-lookup"><span data-stu-id="c2c62-121">Thread-Relative Static Fields</span></span>  
 <span data-ttu-id="c2c62-122">Se você sabe que um dado é sempre exclusivo a uma combinação de thread e domínio de aplicativo, aplique o atributo <xref:System.ThreadStaticAttribute> ao campo estático.</span><span class="sxs-lookup"><span data-stu-id="c2c62-122">If you know that a piece of data is always unique to a thread and application-domain combination, apply the <xref:System.ThreadStaticAttribute> attribute to the static field.</span></span> <span data-ttu-id="c2c62-123">Use o campo como você usaria qualquer outro campo estático.</span><span class="sxs-lookup"><span data-stu-id="c2c62-123">Use the field as you would use any other static field.</span></span> <span data-ttu-id="c2c62-124">Os dados no campo são exclusivos para cada thread que os usa.</span><span class="sxs-lookup"><span data-stu-id="c2c62-124">The data in the field is unique to each thread that uses it.</span></span>  
  
 <span data-ttu-id="c2c62-125">Campos estáticos relativos a thread fornecem melhor desempenho do que os slots de dados e têm o benefício da verificação do tipo de tempo de compilação.</span><span class="sxs-lookup"><span data-stu-id="c2c62-125">Thread-relative static fields provide better performance than data slots and have the benefit of compile-time type checking.</span></span>  
  
 <span data-ttu-id="c2c62-126">Lembre-se de que qualquer código de construtor de classe será executado no primeiro thread no primeiro contexto que acessa o campo.</span><span class="sxs-lookup"><span data-stu-id="c2c62-126">Be aware that any class constructor code will run on the first thread in the first context that accesses the field.</span></span> <span data-ttu-id="c2c62-127">Em todos os outros threads ou contextos no mesmo domínio de aplicativo, os campos serão inicializados para `null` (`Nothing` no Visual Basic) se forem tipos de referência, ou para seus valores padrão se forem tipos de valor.</span><span class="sxs-lookup"><span data-stu-id="c2c62-127">In all other threads or contexts in the same application domain, the fields will be initialized to `null` (`Nothing` in Visual Basic) if they are reference types, or to their default values if they are value types.</span></span> <span data-ttu-id="c2c62-128">Portanto, você não deve confiar em construtores de classe para inicializar campos estáticos relacionados a thread.</span><span class="sxs-lookup"><span data-stu-id="c2c62-128">Therefore, you should not rely on class constructors to initialize thread-relative static fields.</span></span> <span data-ttu-id="c2c62-129">Em vez disso, evite a inicialização de campos estáticos relacionados a thread e suponha que eles sejam inicializados para `null` (`Nothing`) ou para seus valores padrão.</span><span class="sxs-lookup"><span data-stu-id="c2c62-129">Instead, avoid initializing thread-relative static fields and assume that they are initialized to `null` (`Nothing`) or to their default values.</span></span>  
  
## <a name="data-slots"></a><span data-ttu-id="c2c62-130">Slots de dados</span><span class="sxs-lookup"><span data-stu-id="c2c62-130">Data Slots</span></span>  
 <span data-ttu-id="c2c62-131">O .NET Framework fornece slots de dados dinâmicos que são exclusivos a uma combinação de thread e domínio de aplicativo.</span><span class="sxs-lookup"><span data-stu-id="c2c62-131">The .NET Framework provides dynamic data slots that are unique to a combination of thread and application-domain.</span></span> <span data-ttu-id="c2c62-132">Existem dois tipos de slots de dados: slots nomeados e slots sem nome.</span><span class="sxs-lookup"><span data-stu-id="c2c62-132">There are two types of data slots: named slots and unnamed slots.</span></span> <span data-ttu-id="c2c62-133">Ambos são implementados usando a estrutura <xref:System.LocalDataStoreSlot>.</span><span class="sxs-lookup"><span data-stu-id="c2c62-133">Both are implemented by using the <xref:System.LocalDataStoreSlot> structure.</span></span>  
  
-   <span data-ttu-id="c2c62-134">Para criar um slot de dados nomeado, use o método <xref:System.Threading.Thread.AllocateNamedDataSlot%2A?displayProperty=nameWithType> ou <xref:System.Threading.Thread.GetNamedDataSlot%2A?displayProperty=nameWithType>.</span><span class="sxs-lookup"><span data-stu-id="c2c62-134">To create a named data slot, use the <xref:System.Threading.Thread.AllocateNamedDataSlot%2A?displayProperty=nameWithType> or <xref:System.Threading.Thread.GetNamedDataSlot%2A?displayProperty=nameWithType> method.</span></span> <span data-ttu-id="c2c62-135">Para obter uma referência a um slot nomeado existente, passe seu nome para o método <xref:System.Threading.Thread.GetNamedDataSlot%2A>.</span><span class="sxs-lookup"><span data-stu-id="c2c62-135">To get a reference to an existing named slot, pass its name to the <xref:System.Threading.Thread.GetNamedDataSlot%2A> method.</span></span>  
  
-   <span data-ttu-id="c2c62-136">Para criar um slot de dados sem nome, use o método <xref:System.Threading.Thread.AllocateDataSlot%2A?displayProperty=nameWithType>.</span><span class="sxs-lookup"><span data-stu-id="c2c62-136">To create an unnamed data slot, use the <xref:System.Threading.Thread.AllocateDataSlot%2A?displayProperty=nameWithType> method.</span></span>  
  
 <span data-ttu-id="c2c62-137">Para slots nomeados e sem nome, use os métodos <xref:System.Threading.Thread.SetData%2A?displayProperty=nameWithType> e <xref:System.Threading.Thread.GetData%2A?displayProperty=nameWithType> para definir e recuperar as informações no slot.</span><span class="sxs-lookup"><span data-stu-id="c2c62-137">For both named and unnamed slots, use the <xref:System.Threading.Thread.SetData%2A?displayProperty=nameWithType> and <xref:System.Threading.Thread.GetData%2A?displayProperty=nameWithType> methods to set and retrieve the information in the slot.</span></span> <span data-ttu-id="c2c62-138">Estes são métodos estáticos que sempre atuam sobre os dados do thread que os está executando atualmente.</span><span class="sxs-lookup"><span data-stu-id="c2c62-138">These are static methods that always act on the data for the thread that is currently executing them.</span></span>  
  
 <span data-ttu-id="c2c62-139">Os slots nomeados podem ser convenientes, pois você pode recuperar o slot quando necessário, passando seu nome para o método <xref:System.Threading.Thread.GetNamedDataSlot%2A>, em vez de manter uma referência a um slot sem nome.</span><span class="sxs-lookup"><span data-stu-id="c2c62-139">Named slots can be convenient, because you can retrieve the slot when you need it by passing its name to the <xref:System.Threading.Thread.GetNamedDataSlot%2A> method, instead of maintaining a reference to an unnamed slot.</span></span> <span data-ttu-id="c2c62-140">No entanto, se outro componente usa o mesmo nome para seu armazenamento relativo ao thread e um thread executa o código do seu componente e do outro componente, os dois componentes podem corromper os dados uns dos outros.</span><span class="sxs-lookup"><span data-stu-id="c2c62-140">However, if another component uses the same name for its thread-relative storage and a thread executes code from both your component and the other component, the two components might corrupt each other's data.</span></span> <span data-ttu-id="c2c62-141">(Este cenário pressupõe que ambos os componentes estão sendo executados no mesmo domínio de aplicativo e que eles não foram projetados para compartilhar os mesmos dados).</span><span class="sxs-lookup"><span data-stu-id="c2c62-141">(This scenario assumes that both components are running in the same application domain, and that they are not designed to share the same data.)</span></span>
