* <span data-ttu-id="d8454-126">Estruturas</span><span class="sxs-lookup"><span data-stu-id="d8454-126">Structures</span></span>
* <span data-ttu-id="d8454-127">Enums</span><span class="sxs-lookup"><span data-stu-id="d8454-127">Enums</span></span>
* <span data-ttu-id="d8454-128">Interfaces</span><span class="sxs-lookup"><span data-stu-id="d8454-128">Interfaces</span></span>
* <span data-ttu-id="d8454-129">Delegados</span><span class="sxs-lookup"><span data-stu-id="d8454-129">Delegates</span></span>

## <a name="common-language-specification"></a><span data-ttu-id="d8454-132">Common Language Specification</span><span class="sxs-lookup"><span data-stu-id="d8454-132">Common Language Specification</span></span>

<span data-ttu-id="d8454-133">Para habilitar cenários de interoperabilidade completa, todos os objetos que são criados em código devem se basear em alguns pontos em comum nos idiomas que estão consumindo eles (são seus _chamadores_).</span><span class="sxs-lookup"><span data-stu-id="d8454-133">To enable full interoperability scenarios, all objects that are created in code must rely on some commonality in the languages that are consuming them (are their _callers_).</span></span> <span data-ttu-id="d8454-134">Como há várias linguagens diferentes, o .NET especificou essas semelhanças em algo chamado de CLS **(Common Language Specification)**.</span><span class="sxs-lookup"><span data-stu-id="d8454-134">Since there are numerous different languages, .NET has specified those commonalities in something called the **Common Language Specification** (CLS).</span></span> <span data-ttu-id="d8454-135">A CLS define um conjunto de recursos que são necessários para muitos aplicativos comuns.</span><span class="sxs-lookup"><span data-stu-id="d8454-135">CLS defines a set of features that are needed by many common applications.</span></span> <span data-ttu-id="d8454-136">Ela também fornece uma espécie de receita para qualquer linguagem que é implementada no .NET, no que ela precisar dar suporte.</span><span class="sxs-lookup"><span data-stu-id="d8454-136">It also provides a sort of recipe for any language that is implemented on top of .NET on what it needs to support.</span></span>

<span data-ttu-id="d8454-137">A CLS é um subconjunto do CTS.</span><span class="sxs-lookup"><span data-stu-id="d8454-137">CLS is a subset of the CTS.</span></span> <span data-ttu-id="d8454-138">Isso significa que todas as regras no CTS também se aplicam à CLS, a menos que as regras da CLS sejam mais estritas.</span><span class="sxs-lookup"><span data-stu-id="d8454-138">This means that all of the rules in the CTS also apply to the CLS, unless the CLS rules are more strict.</span></span> <span data-ttu-id="d8454-139">Se um componente é criado usando apenas as regras na CLS, ou seja, ele expõe apenas os recursos da CLS em sua API, ele é chamado de **compatível com CLS**.</span><span class="sxs-lookup"><span data-stu-id="d8454-139">If a component is built using only the rules in the CLS, that is, it exposes only the CLS features in its API, it is said to be **CLS-compliant**.</span></span> <span data-ttu-id="d8454-140">Por exemplo, as `<framework-librares>` estão em conformidade com CLS precisamente porque precisam funcionar em todas as linguagens com suporte no .NET.</span><span class="sxs-lookup"><span data-stu-id="d8454-140">For instance, the `<framework-librares>` are CLS-compliant precisely because they need to work across all of the languages that are supported on .NET.</span></span>
