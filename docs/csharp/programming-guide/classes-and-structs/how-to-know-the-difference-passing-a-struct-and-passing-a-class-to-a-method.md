 <span data-ttu-id="05781-108">Como um struct é um [tipo de valor](../../language-reference/builtin-types/value-types.md), ao [passar um struct por valor](./passing-value-type-parameters.md) a um método, esse método receberá e operará em uma cópia do argumento do struct.</span><span class="sxs-lookup"><span data-stu-id="05781-108">Because a struct is a [value type](../../language-reference/builtin-types/value-types.md), when you [pass a struct by value](./passing-value-type-parameters.md) to a method, the method receives and operates on a copy of the struct argument.</span></span> <span data-ttu-id="05781-109">O método não tem acesso ao struct original no método de chamada e, portanto, não é possível alterá-lo de forma alguma.</span><span class="sxs-lookup"><span data-stu-id="05781-109">The method has no access to the original struct in the calling method and therefore can't change it in any way.</span></span> <span data-ttu-id="05781-110">O método pode alterar somente a cópia.</span><span class="sxs-lookup"><span data-stu-id="05781-110">The method can change only the copy.</span></span>  
  
 <span data-ttu-id="05781-111">Uma instância de classe é um [tipo de referência](../../language-reference/keywords/reference-types.md) e não é um tipo de valor.</span><span class="sxs-lookup"><span the calling method.</span></span> <span data-ttu-id="05781-116">No entanto, o método chamado pode usar a cópia do endereço para acessar os membros da classe que o endereço original e a cópia da será alterada.</span><span class="sxs-lookup"><span data-stu-id="05781-117">If the called method changes a class member, the original class instance in the calling method also changes.</span></span>
