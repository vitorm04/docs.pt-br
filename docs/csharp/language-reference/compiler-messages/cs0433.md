 <span data-ttu-id="3e793-104">Dois assemblies diferentes referenciados no aplicativo contêm o mesmo namespace e tipo, o que gera ambiguidade.</span><span class="sxs-lookup"><span data-stu-id="3e793-104">Two different assemblies referenced in your application contain the same namespace and type, which produces ambiguity.</span></span>  
  
 <span data-ttu-id="3e793-105">Para resolver esse erro, use o recurso de alias da opção do compilador [/reference (Opções do Compilador C#)](../../../csharp/language-reference/compiler-options/reference-compiler-option.md) ou não referencie um dos assemblies.</span><span class="sxs-lookup"><span data-stu-id="3e793-105">To resolve this error, use the alias feature of the [/reference (C# Compiler Options)](../../../csharp/language-reference/compiler-options/reference-compiler-option.md) compiler option or do not reference one of your assemblies.</span></span>

## <a name="example"></a><span data-ttu-id="3e793-110">Exemplo</span><span class="sxs-lookup"><span data-stu-id="3e793-110">Example</span></span>  
 <span data-ttu-id="3e793-111">O exemplo a seguir gera o erro CS0433.</span><span class="sxs-lookup"><span data-stu-id="3e793-111">The following example generates CS0433.</span></span>  
  
```csharp

using TypeBindConflicts;  
public class Test   
{  
   public static void Main()   
   {  
      AggPubImpAggPubImp n6 = new AggPubImpAggPubImp();   // CS0433  
   }  
}  
```  
  
## <a name="example"></a><span data-ttu-id="3e793-112">Exemplo</span><span class="sxs-lookup"><span data-stu-id="3e793-112">Example</span></span>
