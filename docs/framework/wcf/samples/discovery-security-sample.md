## <a name="secure-channel-factory"></a><span data-ttu-id="f7258-116">Fábrica de canal seguro</span><span class="sxs-lookup"><span data-stu-id="f7258-116">Secure Channel Factory</span></span>  
 <span data-ttu-id="f7258-117">A fábrica de canal seguro cria canais de saída ou duplex que adicionam uma assinatura compacta a cabeçalhos de mensagens.</span><span class="sxs-lookup"><span data-stu-id="f7258-117">The secure channel factory creates output or duplex channels that add a compact signature to message headers.</span></span> <span data-ttu-id="f7258-118">Para manter as mensagens tão pequenas quanto possível o formato de assinatura compacta é usado.</span><span class="sxs-lookup"><span data-stu-id="f7258-118">To keep messages as small as possible the compact signature format is used.</span></span> <span data-ttu-id="f7258-119">A estrutura de uma assinatura compacta é mostrada no exemplo a seguir.</span><span class="sxs-lookup"><span data-stu-id="f7258-119">The structure of a compact signature is shown in the following example.</span></span>  
  
```xml  
<d:Security ... >   
  [<d:Sig Scheme="xs:anyURI"   
         [KeyId="xs:base64Binary"]?  
          Refs="..."  
         [PrefixList]="xs:NMTOKENS"   
          Sig="xs:base64Binary"   
          ... />]?  
  ...   
</d:Security>  
```  
  
> [!NOTE]
> <span data-ttu-id="f7258-120">O `PrefixList` foi adicionado no protocolo de versão de descoberta 2008.</span><span class="sxs-lookup"><span data-stu-id="f7258-120">The `PrefixList` was added in the 2008 Discovery version protocol.</span></span>  
  
 <span data-ttu-id="f7258-121">Para calcular a assinatura, o exemplo determina os itens de assinatura expandidos.</span><span class="sxs-lookup"><span data-stu-id="f7258-121">To compute the signature, the sample determines the expanded signature items.</span></span> <span data-ttu-id="f7258-122">Uma assinatura XML (`SignedInfo`) é criada, usando o `ds` prefixo de namespace, conforme exigido pela especificação WS-Discovery.</span><span class="sxs-lookup"><span data-stu-id="f7258-122">An XML signature (`SignedInfo`) is created, using the `ds` namespace prefix, as required by the WS-Discovery specification.</span></span> <span data-ttu-id="f7258-123">O corpo e todos os cabeçalhos nos namespaces de descoberta e endereçamento são referenciados na assinatura, para que eles não possam ser adulterados.</span><span class="sxs-lookup"><span data-stu-id="f7258-123">The body and all the headers in discovery and addressing namespaces are referenced in the signature, so they cannot be tampered with.</span></span> <span data-ttu-id="f7258-124">Cada elemento referenciado é transformado usando a canonicalização exclusiva (http://www.w3.org/2001/10/xml-exc-c14n# ) e, em seguida, um valor de Resumo de SHA-1 é computado (http://www.w3.org/2000/09/xmldsig#sha1 ).</span><span class="sxs-lookup"><span data-stu-id="f7258-124">Each referenced element is transformed using the Exclusive Canonicalization (http://www.w3.org/2001/10/xml-exc-c14n# ), and then an SHA-1 digest value is computed (http://www.w3.org/2000/09/xmldsig#sha1 ).</span></span> <span data-ttu-id="f7258-125">Com base em todos os elementos referenciados e seus valores de resumo, o valor da assinatura é calculado usando http://www.w3.org/2000/09/xmldsig#rsa-sha1 o algoritmo RSA ().</span><span class="sxs-lookup"><span data-stu-id="f7258-125">Based on all referenced elements and their digest values, the signature value is computed using the RSA algorithm (http://www.w3.org/2000/09/xmldsig#rsa-sha1 ).</span></span>  
  
 <span data-ttu-id="f7258-126">As mensagens são assinadas com um certificado especificado pelo cliente.</span><span class="sxs-lookup"><span data-stu-id="f7258-126">The messages are signed with a client-specified certificate.</span></span> <span data-ttu-id="f7258-127">O local do repositório, o nome e o nome da entidade do certificado devem ser especificados quando o elemento de associação é criado.</span><span class="sxs-lookup"><span data-stu-id="f7258-127">The store location, name and the certificate subject name must be specified when the binding element is created.</span></span> <span data-ttu-id="f7258-128">O `KeyId` na assinatura compacta representa o identificador de chave do token de assinatura e é o identificador de chave da entidade (esqui) do token de assinatura ou (se a esqui não existir) um hash SHA-1 da chave pública do token de assinatura.</span><span class="sxs-lookup"><span data-stu-id="f7258-128">The `KeyId` in the compact signature represents the key identifier of the signing token and is the Subject Key Identifier (SKI) of the signing token or (if the SKI does not exist) a SHA-1 hash of the public key of the signing token.</span></span>
