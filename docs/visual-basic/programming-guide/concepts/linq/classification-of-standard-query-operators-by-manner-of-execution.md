<span data-ttu-id="deb9b-103">As implementações de LINQ to Objects dos métodos de operador de consulta padrão em uma das duas maneiras principais: imediata ou adiada.</span><span class="sxs-lookup"><span data-stu-id="deb9b-103">The LINQ to Objects implementations of the standard query operator methods execute in one of two main ways: immediate or deferred.</span></span> <span data-ttu-id="deb9b-104">Os operadores de consulta que usam a execução adiada podem ser divididos em mais duas categorias: streaming e não streaming.</span><span class="sxs-lookup"><span data-stu-id="deb9b-104">The query operators that use deferred execution can be additionally divided into two categories: streaming and non-streaming.</span></span> <span data-ttu-id="deb9b-105">Se você souber como os operadores de consulta diferentes são executados, isso poderá ajudá-lo a entender os resultados que serão obtidos de uma determinada consulta.</span><span class="sxs-lookup"><span data-stu-id="deb9b-105">If you know how the different query operators execute, it may help you understand the results that you get from a given query.</span></span> <span data-ttu-id="deb9b-106">Isso é especialmente verdadeiro se a fonte de dados está sendo alterada ou se você estiver criando uma consulta sobre outra consulta.</span><span class="sxs-lookup"><span data-stu-id="deb9b-106">This is especially true if the data source is changing or if you are building a query on top of another query.</span></span> <span data-ttu-id="deb9b-107">Este tópico classifica os operadores de consulta padrão de acordo com o modo de execução.</span><span class="sxs-lookup"><span data-stu-id="deb9b-107">This topic classifies the standard query operators according to their manner of execution.</span></span>  
  
## <a name="manners-of-execution"></a><span data-ttu-id="deb9b-108">Modos de execução</span><span class="sxs-lookup"><span data-stu-id="deb9b-108">Manners of Execution</span></span>  
  
### <a name="immediate"></a><span data-ttu-id="deb9b-109">Imediata</span><span class="sxs-lookup"><span data-stu-id="deb9b-109">Immediate</span></span>  
 <span data-ttu-id="deb9b-110">A execução imediata significa que a fonte de dados é lida e a operação é realizada no ponto do código em que a consulta é declarada.</span><span class="sxs-lookup"><span data-stu-id="deb9b-110">Immediate execution means that the data source is read and the operation is performed at the point in the code where the query is declared.</span></span> <span data-ttu-id="deb9b-111">Todos os operadores de consulta padrão que retornam um resultado único e não enumerável são executados imediatamente.</span><span class="sxs-lookup"><span data-stu-id="deb9b-111">All the standard query operators that return a single, non-enumerable result execute immediately.</span></span>  
  
### <a name="deferred"></a><span data-ttu-id="deb9b-112">Adiado</span><span class="sxs-lookup"><span data-stu-id="deb9b-112">Deferred</span></span>  
 <span data-ttu-id="deb9b-113">A execução adiada significa que a operação não será realizada no ponto do código em que a consulta estiver declarada.</span><span class="sxs-lookup"><span data-stu-id="deb9b-113">Deferred execution means that the operation is not performed at the point in the code where the query is declared.</span></span> <span data-ttu-id="deb9b-114">A operação será realizada somente quando a variável de consulta for enumerada, por exemplo, usando uma instrução `For Each`.</span><span class="sxs-lookup"><span data-stu-id="deb9b-114">The operation is performed only when the query variable is enumerated, for example by using a `For Each` statement.</span></span> <span data-ttu-id="deb9b-115">Isso significa que os resultados da execução da consulta dependerão do conteúdo da fonte de dados quando a consulta for executada em vez de quando a consulta for definida.</span><span class="sxs-lookup"><span data-stu-id="deb9b-115">This means that the results of executing the query depend on the contents of the data source when the query is executed rather than when the query is defined.</span></span> <span data-ttu-id="deb9b-116">Se a variável de consulta for enumerada várias vezes, os resultados poderão ser diferentes a cada vez.</span><span class="sxs-lookup"><span data-stu-id="deb9b-116">If the query variable is enumerated multiple times, the results might differ every time.</span></span> <span data-ttu-id="deb9b-117">Quase todos os operadores de consulta padrão cujo tipo de retorno é <xref:System.Collections.Generic.IEnumerable%601> ou <xref:System.Linq.IOrderedEnumerable%601> executam de maneira adiada.</span><span class="sxs-lookup"><span data-stu-id="deb9b-117">Almost all the standard query operators whose return type is <xref:System.Collections.Generic.IEnumerable%601> or <xref:System.Linq.IOrderedEnumerable%601> execute in a deferred manner.</span></span>  
  
 <span data-ttu-id="deb9b-118">Os operadores de consulta que usam a execução adiada podem ser, adicionalmente, classificados como streaming ou não streaming.</span><span class="sxs-lookup"><span data-stu-id="deb9b-118">Query operators that use deferred execution can be additionally classified as streaming or non-streaming.</span></span>  
  
#### <a name="streaming"></a><span data-ttu-id="deb9b-119">Streaming</span><span class="sxs-lookup"><span data-stu-id="deb9b-119">Streaming</span></span>  
 <span data-ttu-id="deb9b-120">Operadores streaming não precisam ler todos os dados de origem antes de gerar elementos.</span><span class="sxs-lookup"><span data-stu-id="deb9b-120">Streaming operators do not have to read all the source data before they yield elements.</span></span> <span data-ttu-id="deb9b-121">No momento da execução, um operador streaming realiza sua operação em cada elemento de origem enquanto eles são lidos, gerando o elemento, se apropriado.</span><span class="sxs-lookup"><span data-stu-id="deb9b-121">At the time of execution, a streaming operator performs its operation on each source element as it is read and yields the element if appropriate.</span></span> <span data-ttu-id="deb9b-122">Um operador streaming continua a ler os elementos de origem até que um elemento de resultado possa ser produzido.</span><span class="sxs-lookup"><span data-stu-id="deb9b-122">A streaming operator continues to read source elements until a result element can be produced.</span></span> <span data-ttu-id="deb9b-123">Isso significa que mais de um elemento de origem poderá ser lido para produzir um elemento de resultado.</span><span class="sxs-lookup"><span data-stu-id="deb9b-123">This means that more than one source element might be read to produce one result element.</span></span>  
  
#### <a name="non-streaming"></a><span data-ttu-id="deb9b-124">Não streaming</span><span class="sxs-lookup"><span data-stu-id="deb9b-124">Non-Streaming</span></span>  
 <span data-ttu-id="deb9b-125">Os operadores não streaming devem ler todos os dados de origem antes de produzirem um elemento de resultado.</span><span class="sxs-lookup"><span data-stu-id="deb9b-125">Non-streaming operators must read all the source data before they can yield a result element.</span></span> <span data-ttu-id="deb9b-126">Operações como classificação ou agrupamento se enquadram nesta categoria.</span><span class="sxs-lookup"><span data-stu-id="deb9b-126">Operations such as sorting or grouping fall into this category.</span></span> <span data-ttu-id="deb9b-127">No momento da execução, os operadores de consulta não streaming leem todos os dados de origem, colocam-nos em uma estrutura de dados, realizam a operação e geram os elementos de resultado.</span><span class="sxs-lookup"><span data-stu-id="deb9b-127">At the time of execution, non-streaming query operators read all the source data, put it into a data structure, perform the operation, and yield the resulting elements.</span></span>
